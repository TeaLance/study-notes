Chapter 3

Arithmetic for Computers 算術運算

電腦的本質是「會算數的開關」。這一章從「數字怎麼用位元表示」開始，一路把加減、比較、乘除、 浮點數做成硬體。核心問題只有兩個：怎麼算得對（溢位、有號數）與 怎麼算得快（進位預看、Booth）。

3.1有號數與二補數

電腦用固定位元數存整數。二補數 (two's complement) 是最常見的有號表示法：最高位是符號位， 負數 = 把正數每位元反相後 +1。它的好處是加減法用同一套加法器就能做，且 0 只有一種表示。

  • 範圍：n 位元可表示 −2n−1 到 2n−1−1（8 位元為 −128～127）。
  • 符號延伸 (sign extension)：把短數字放進長暫存器時，複製符號位填滿高位，數值不變（lb、addi 都會用到）。
  • 取負：−x = ~x + 1。這也是減法 A − B = A + (~B) + 1 的由來。
詳解：為什麼「反相加一」就是負數？

二補數的定義是把最高位的權重設成負的：對 8 位元，值 = −b7·27 + b6·26 + … + b0·20。

觀察 x + ~x = 1111…1 = −1（全部是 1 的二補數正好是 −1）。移項得 ~x = −1 − x，所以 ~x + 1 = −x。這就是取負的公式，完全不需要特別的「減法器」。

下面的工具用 8 位元二補數做加法，逐位顯示進位，並判斷是否溢位。

3.2加減法與溢位偵測

溢位 (overflow) 指運算結果超出可表示範圍。重點：只有同號相加、或異號相減才可能溢位； 無號數則看最高位是否進位。硬體判斷法：進入最高位的進位 ≠ 離開最高位的進位 即溢位。

運算運算元 A,B 符號結果符號異常時
A + BA≥0, B≥0結果 < 0 → 溢位
A + BA<0, B<0結果 ≥ 0 → 溢位
A − BA≥0, B<0結果 < 0 → 溢位
A − BA<0, B≥0結果 ≥ 0 → 溢位
MIPS：add / sub / addi 會偵測溢位並丟例外；addu / subu / addiu 不偵測。所以 C 編譯器多半用 u 版本。

互動：二補數加法 / 溢位視覺化

3.3從 1-bit ALU 到 32-bit ALU

ALU (Arithmetic Logic Unit) 是 CPU 的計算核心。先做出 1 位元 ALU（含 AND、OR、全加器，再用 多工器 (MUX) 選輸出），把 32 個串起來就是 32 位元 ALU。

  • 全加器 (full adder)：3 進 2 出。Sum = A⊕B⊕Cin，Cout = AB + (A⊕B)·Cin。
  • 減法：把 B 反相（Binvert=1）並設最低位 CarryIn=1，等於做 A + ~B + 1。
  • slt (set less than)：用 A − B 的符號位（最高位）回送到最低位的 Less 輸入，結果是 0 或 1。
  • 相等比較 (beq)：算 A − B，用 NOR 把所有結果位元 OR 起來；全 0 代表相等（Zero 訊號）。
詳解：一個 1-bit ALU 裡有什麼？

每個 1-bit ALU 接收 a、b、CarryIn 與一組控制線：

  • Ainvert / Binvert：決定是否反相輸入（做 NAND/NOR/減法）。
  • Operation (2 bit)：MUX 選擇要輸出 AND(00)、OR(01)、Add(10) 還是 Less(11)。

32 顆串成漣波，最低位的 CarryIn 接 Binvert（減法時 = 1），最高位的 ALU 多輸出一條 Set 給 slt，以及 Overflow 偵測。整顆 ALU 的控制線合稱 ALU control (4 bit)，由 Chapter 4 的控制單元產生。

3.4進位預看加法器 Carry Lookahead

把 32 顆 1-bit 加法器直接串起來叫 漣波進位 (ripple-carry)：進位要一位一位傳，最長路徑是 n 倍閘延遲——太慢。 Carry Lookahead Adder (CLA) 用「先算進位」的技巧把延遲壓到 O(log n) 等級。

generate　gi = ai · bi　　（兩個都是 1，一定產生進位）
propagate pi = ai + bi　（至少一個是 1，會把進位往上傳）

c1 = g0 + p0·c0
c2 = g1 + p1g0 + p1p0c0
c3 = g2 + p2g1 + p2p1g0 + p2p1p0c0
c4 = …（每個進位都只用 g、p、c0 直接算，不必等前一級）
  • 兩層架構：4-bit CLA 為一塊，再用區塊級的 G、P 把多塊組成 16-bit / 32-bit，形成階層式預看。
  • 區塊 generate：G = g3 + p3g2 + p3p2g1 + p3p2p1g0，區塊 P = p3p2p1p0。
  • 取捨：CLA 快但邏輯閘多、佈線複雜；實務常用混合（局部 CLA + 區塊漣波）。

3.5乘法器 Multiplication

紙筆乘法 = 對每個乘數位元，決定要不要加上被乘數，然後左移。硬體把這流程做成「加 → 位移 → 重複」的迴圈。 課程演進三個版本，逐步把浪費的硬體砍掉：

版本做法改良
Ver 1被乘數左移、64-bit ALU、Product 64-bit直觀但一半 ALU 在算 0，浪費
Ver 2被乘數不動，改成 Product 右移、32-bit ALUALU 縮一半
Ver 3乘數塞進 Product 低位，省掉乘數暫存器對應 MIPS 的 Hi/Lo

每一輪：① 測 Product 最低位，是 1 就把被乘數加到 Product 高半部；② Product 整體右移 1 位。重複 n 次。

3.7除法與 IEEE-754 浮點數

除法與乘法對稱：用「減 → 位移 → 看餘數正負決定上商」的迴圈（restoring / non-restoring division）， MIPS 的商與餘數一樣放在 Hi/Lo。

IEEE-754 是浮點數的標準格式：(−1)S × 1.M × 2(E−bias)。 指數用 bias 表示（單精度 127、雙精度 1023），尾數隱含一個「1.」。

Chapter 4

The Processor 處理器

有了會算數的 ALU，接著要把記憶體、暫存器、ALU 接成能自動執行指令的處理器。 先學最直覺的「單週期」設計，再用「管線」把效能拉高——而管線帶來的麻煩（危障）正是本章重頭戲。

4.1指令執行的五大步驟

不論哪種指令，MIPS 都可拆成五個階段，這也正是後面五級管線的由來：

  • IF (Instruction Fetch)：用 PC 從指令記憶體取指令，PC ← PC + 4。
  • ID (Instruction Decode)：解碼，讀暫存器檔的兩個來源暫存器，符號延伸 immediate。
  • EX (Execute)：ALU 運算——算結果 / 算記憶體位址 / 算分支目標。
  • MEM (Memory access)：只有 lw / sw 會存取資料記憶體。
  • WB (Write Back)：把結果寫回暫存器檔。
三類指令：算術邏輯 add, sub, and, or, slt；記憶體 lw, sw；控制轉移 beq, j。它們走的路徑不同，但共用同一組硬體。

把上述零件用線接起來就是資料路徑 (datapath)。單週期 = 每條指令在一個時脈週期內走完全程。 下面選不同指令，看它實際用到哪些零件與線路（亮起 = 啟用），右側對應的控制訊號也會跟著變。

4.3控制訊號 Control Signals

控制單元 (control unit) 解碼指令的 opcode，產生上面那排控制線去「指揮」MUX、記憶體與暫存器讀寫。常見訊號：

  • RegDst：寫回的暫存器編號來自 rt(0) 還是 rd(1)。
  • ALUSrc：ALU 第二運算元來自暫存器(0) 還是符號延伸的 immediate(1)。
  • MemRead / MemWrite：是否讀 / 寫資料記憶體。
  • MemtoReg：寫回的值來自 ALU(0) 還是資料記憶體(1)。
  • RegWrite：是否寫回暫存器；Branch：是否為分支（搭配 Zero 決定是否跳）。
  • ALUOp：兩位元，配合 funct 欄位由 ALU control 產生真正的 4-bit ALU 控制。

4.4管線化 Pipelining

單週期的缺點：時脈被最慢的指令（lw）綁死。管線 (pipeline) 像洗衣店流水線—— 把指令切成五級，讓不同指令的不同階段同時進行。它不縮短單一指令的時間，而是大幅提高吞吐量 (throughput)。

理想加速比 ≈ 管線級數（這裡 = 5）
條件：各級時間均衡、且沒有危障 (hazard) 造成停頓
  • 級與級之間插入管線暫存器（IF/ID、ID/EX、EX/MEM、MEM/WB）保存中間狀態。
  • 穩定狀態下每個時脈完成一條指令（CPI → 1），但啟動有「填管線」的延遲。

4.5危障 Hazards 與前送 Forwarding

管線重疊執行會產生危障 (hazard)——三種：

  • 結構危障 (structural)：硬體資源不夠用（例如同一個記憶體同時要取指令又要讀資料）。解法：指令/資料記憶體分開。
  • 資料危障 (data)：後面指令要用前面還沒寫回的結果。解法：前送 (forwarding) 把 ALU 結果直接送回去；lw 之後仍需 1 個 停頓 (stall / bubble)。
  • 控制危障 (control)：分支要等條件算完才知道跳不跳。解法：分支預測 (branch prediction)、提早判斷分支。

Chapter 5

Memory Hierarchy 記憶體階層

處理器越來越快，記憶體卻沒跟上（CPU-DRAM gap）。解法不是做一個又快又大又便宜的記憶體（不存在）， 而是疊出一個階層：用小而快的擋在前面，大而慢的放後面，靠局部性讓多數存取都打中快的那層。

5.1局部性原理 Principle of Locality

程式的記憶體存取不是隨機的，而有局部性 (locality)——這是整個記憶體階層能成立的關鍵：

  • 時間局部性 (temporal)：剛用過的東西，很可能馬上又用（迴圈變數、指令）。
  • 空間局部性 (spatial)：用了某位址，附近的位址也很可能被用（陣列、循序指令）。

因此把資料以區塊 (block / line) 為單位，從慢的層搬到快的層，就能用少量快記憶體服務大多數存取。

點選任一層查看速度、容量與成本特性。

5.2記憶體技術 SRAM / DRAM / Disk

技術原理速度每 GB 成本用途
SRAM正反器（6 顆電晶體/bit），不需更新最快 (~1ns)最貴Cache
DRAM電容存電荷（1T/bit），需週期性 refresh中 (~50ns)中主記憶體

  • DRAM 以列 (row) 為單位讀取整列；DDR / QDR 在時脈邊緣多次傳資料來提高頻寬。

5.3Cache 基礎與對映方式

Cache 是 CPU 與主記憶體之間的小型 SRAM。一次存取若在 cache 找到叫 命中 (hit)，找不到叫 失誤 (miss)，要從下層搬一個區塊上來（付出 miss penalty）。

hit ratio = hits / accesses　　miss ratio = 1 − hit ratio
平均存取時間 AMAT = hit time + miss ratio × miss penalty

位址被切成三段來定位：

［　Tag　｜　Index（選哪一行）｜　Offset（區塊內位元組）　］
  • 直接對映 (direct-mapped)：每個記憶體區塊只能放在固定一行（index = 區塊號 mod 行數）。簡單但易衝突。
  • 組相聯 (set-associative)：每組有 N 行，區塊可放組內任一行，衝突變少。
  • 全相聯 (fully-associative)：可放任意行，最少衝突但比對成本最高。
  • 寫入策略：write-through（同時寫下層）vs write-back（只寫 cache，靠 dirty bit 標記，換出時才寫回）；valid bit 標記該行是否有效。

5.4虛擬記憶體 Virtual Memory 與 TLB

虛擬記憶體 把「主記憶體當成磁碟的 cache」：每個程式以為自己擁有完整連續的位址空間， 硬體 + OS 再把虛擬位址翻成實體位址。單位是 page（通常 4KB）。

  • 頁表 (page table)：記錄每個虛擬頁對應的實體頁框；找不到（不在記憶體）叫 page fault，要從磁碟搬入。
  • TLB (Translation Lookaside Buffer)：頁表的 cache。每次存取要先翻譯位址，若每次都查頁表（在記憶體）太慢，TLB 用小型快取存最近的翻譯。
  • 對應關係：cache 的 hit/miss ↔ TLB hit/miss ↔ page fault，都是「上層找不到就往下層拿」的同一個思想。
一條 load 的完整路徑：虛擬位址 → 查 TLB（翻成實體位址）→ 查 cache → 命中就拿資料，失誤就往主記憶體 / 磁碟一路追下去。整個 Chapter 5 就是這條鏈上每一環的最佳化。
